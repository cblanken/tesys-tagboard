How to contribute
=================

Tesy's Tagboard is open to contributions, but please be aware that the project is maintained by a small number of developers in their free time. Please read through the :doc:`development` guide to get started if you'd like to jump right in. Otherwise you might be interested in reading through the :doc:`../architecture/index` breakdown to get a better idea of how everything fits together.

If you aren't able to contribute code, **don't worry!** There are plenty of other ways you may be able to contribute.

* First and foremost, bug reports help immensely to find where the application may be deficient or failing. If you find a bug please don't hesitate to `report an issue <https://github.com/cblanken/tesys_tagboard/issues/new>`__

* Verifying existing `bug reports <https://github.com/cblanken/tesys-tagboard/issues>`_ is also very helpful. It indicates to us that the bug is reproducible and isn't some one-off configuration problem. Documenting any reproduction steps will help us triage and resolve issues more quickly.
